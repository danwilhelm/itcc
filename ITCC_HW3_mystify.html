Challenge #2. Let's put your new library to use by making a screen saver (well, as close to one as we can get without installing it)! One of the most popular screen savers that comes with the Windows OS is Mystify. Let's try to emulate this by using our line functions and our knowledge of arrays, except let's make it a little less complicated by only making each quadrilateral consist of ONE line instead of four. Some things to note:

  • Use arrays. Maybe we can use arrays to store the point locations and their velocities. Note that each quadrilateral is made up of FOUR points and that lines connect each of these points together. Thus, perhaps every group of four points in our array could be connected by lines ...
  • For an easier version, use rectangles and only store two points per quadrilateral.
  • 45 degree bounces. Remember pong? Make all of your points bounce at 45 degree angles off the sides of the screen. See the Lecture 4 slides for help.
  • rand() for randomness. How would we generate random starting positions for the points? We can use the rand() function, which returns a random number between 0 and 1. srand() seeds the random number generator with a value, making the number generated more random than before. Only use srand() once in your program when your software initializes.

    srand(GetTickCount()); // seeds the generator with the ms since Windows started
    ...
    int rndNum = rand()%640; // generates a random number from 0-639, inclusive.
